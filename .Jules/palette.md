## 2024-05-15 - [DJ Chat Accessibility]
**Learning:** Icon-only floating action buttons (like the AI DJ microphone button) often lack inherent semantic meaning for screen readers, leading to poor accessibility. Similarly, disabled buttons that lack visual indicators or explanatory tooltips cause frustration because users don't know *why* the button is disabled (e.g., waiting on an API vs empty input).
**Action:** Always pair `disabled` logic with explicit visual cues (e.g., `disabled:opacity-50`, `disabled:cursor-not-allowed`) and provide dynamic `title` tooltips that reflect the exact condition causing the disabled state. Consistently apply `aria-label` and `title` to all icon-only buttons (like FABs or close buttons) to provide clear context for assistive technologies and mouse hover users. Ensure keyboard accessibility with explicit `focus-visible` outlines.
