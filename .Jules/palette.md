## 2025-03-01 - Multi-Condition Disabled Buttons
**Learning:** When buttons have multi-condition disabled logic (e.g., missing tracks vs. currently generating), a generic disabled state with a standard "Disabled" tooltip or single `cursor-not-allowed` pointer is insufficient for good UX. It leaves users guessing why the action is blocked. Applying global disabled pointer states via `disabled:cursor-not-allowed` can also cause CSS specificity conflicts with loading states (`cursor-wait`).
**Action:** Always use conditional logic to ensure the `title` tooltip explicitly mirrors the exact failing condition. Apply cursor utility classes conditionally (e.g., `cursor-wait` for generating vs `cursor-not-allowed` for missing inputs) to avoid CSS specificity conflicts and ensure the pointer accurately reflects the system state.
