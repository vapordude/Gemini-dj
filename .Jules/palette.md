## 2024-05-19 - Improved disabled state clarity for the Transition button
**Learning:** In complex web applications like a DJ mixer where state depends on multiple conditions (e.g., both decks loaded and not currently generating), simply adding a `disabled` attribute to a button is insufficient for a good user experience. Users need to understand *why* the button is disabled. Without visual cues (`opacity-50`, `cursor-not-allowed`) and explanatory tooltips (`title`), users might assume the application is broken or unresponsive. Furthermore, ensuring keyboard accessibility by adding explicit focus states (`focus-visible:ring-2 focus-visible:ring-indigo-400 focus-visible:outline-none`) is crucial for non-mouse users.
**Action:** Always pair `disabled` attributes with clear visual cues and explanatory tooltips, especially when the disabled state relies on complex business logic. Ensure all interactive elements have explicit focus indicators for keyboard navigation.
