## 2024-03-21 - Custom Input Accessible Focus within Neon Design
**Learning:** Customizing native `<input type="range">` elements by making them invisible (`opacity-0`) and relying entirely on adjacent custom styled elements (`absolute`, pointer-event-none layers) removes default browser focus rings. To maintain accessibility in these stylized elements without breaking the neon/glassmorphism design, we must use `peer` utility classes on the invisible input to conditionally style the visible adjacent cap when focused via keyboard (`peer-focus-visible:ring-2`).
**Action:** Always check custom stylized inputs where the actual input element is hidden or opacity 0. Add `aria-label` and `focus:outline-none` to the hidden input, use Tailwind's `peer` on the input, and style an adjacent visible element with `peer-focus-visible:ring-2 peer-focus-visible:ring-color` (e.g. `emerald-500` to match existing neon accents).
