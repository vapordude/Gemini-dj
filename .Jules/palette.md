## 2024-04-14 - Conditional Tooltips on Disabled Action Buttons
**Learning:** For application-critical action buttons (like the manual transition button in App.tsx) that can be disabled due to multiple distinct conditions (e.g., waiting on asynchronous generation vs. missing required input state like unassigned tracks), users require specific, tailored explanations. A generic "Disabled" visual cue or tooltip fails to inform the user about what specific action they need to take to unblock themselves.
**Action:** When buttons have multi-condition disabled states, use conditional logic inside the `title` attribute to mirror the exact failing condition (e.g., "Transition currently generating" vs. "Please load tracks on both decks first"). Additionally, differentiate the visual cursor state (e.g., `cursor-wait` for processing, `cursor-not-allowed` with `opacity-50` for missing prerequisites) to provide an extra layer of immediate visual feedback.
