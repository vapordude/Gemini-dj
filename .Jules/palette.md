## 2024-05-18 - Missing Disabled State Explanations for Buttons with Multiple Conditions
**Learning:** Found an accessibility and UX issue in `src/App.tsx` on the "Transition" button, which could be disabled for two different reasons: AI generating a transition OR missing tracks on the decks. Originally, there was no visual indication or tooltip explaining *why* the button was disabled when missing tracks, though a "GENERATING..." state was shown for the former. Screen reader users would have no context. Using `disabled` alone is insufficient when there are multiple potential failure conditions.
**Action:** Applied specific `title` attributes that use conditional logic to explain exactly *why* the button is disabled. Applied visual `opacity-50 cursor-not-allowed` styles so users intuitively know it is disabled *before* attempting interaction. Next time, always check buttons with multi-condition `disabled` logic to ensure both visual styling and screen reader accessible `title`/ARIA attributes accurately reflect the failing condition.
